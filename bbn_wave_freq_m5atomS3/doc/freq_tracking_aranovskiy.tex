\section{Introduction}

This document describes the \emph{implemented} Aranovskiy-style frequency estimator in the provided C\texttt{++} code, rather than an idealized continuous-time observer. The implementation is inspired by the adaptive sinusoidal frequency estimation approach of Bobtsov, Nikolaev, Slita, Borgul, and Aranovskiy~\cite{BobtsovNikolaevSlitaBorgulAranovskiy2013}, but it also includes several practical engineering modifications:
\begin{itemize}
  \item a first-order low-pass state rather than a second-order oscillator state,
  \item explicit forward-Euler discretization,
  \item internal time scaling for low-frequency numerical conditioning,
  \item signal-energy normalization of the adaptation law,
  \item hard clipping of the adaptation rate,
  \item heuristic confidence and lock outputs.
\end{itemize}

Accordingly, this note aims to match the actual source code line-for-line at the algorithm level.

\section{Signal Model and Estimation Objective}

The measured scalar signal is modeled as
\begin{equation}
  y(t) = A(t)\,\sin\bigl(\omega(t) t + \phi(t)\bigr) + \nu(t),
\end{equation}
where
\begin{itemize}
  \item $A(t)$ is a possibly slowly varying amplitude,
  \item $\omega(t)$ is the unknown instantaneous angular frequency,
  \item $\phi(t)$ is the unknown phase,
  \item $\nu(t)$ is measurement noise.
\end{itemize}

The implementation does not estimate $A(t)$ explicitly. Instead, it maintains a filtered state $x_1$ and an adaptive scalar parameter $\sigma$, from which an algebraic parameter $\theta$ is formed. The reported physical frequency is obtained from the magnitude of $\theta$.

\section{Implemented Observer State}

\subsection{First-Order Low-Pass Filter State}

The code implements a \textbf{first-order} dynamic state
\begin{equation}\label{eq:x1_ode}
  \dot{x}_1 = -a\,x_1 + b\,y,
\end{equation}
where $a>0$ and $b>0$ are fixed gains set by the constructor through \texttt{setParams()}. There is no independently integrated second state such as $x_2 = \dot{x}_1$.

Equation~\eqref{eq:x1_ode} is a first-order low-pass filter with time constant $1/a$ and DC gain $b/a$. In the common configuration used by the code, $a=b$, so the DC gain is unity.

\subsection{Adaptive State and Algebraic Parameter}

The implementation stores an adaptive integrator state $\sigma$ and forms the algebraic quantity
\begin{equation}\label{eq:theta_alg}
  \theta = \sigma + k b x_1 y,
\end{equation}
where $k>0$ is the adaptation gain.

The code then maps $\theta$ to an internal angular-frequency magnitude through
\begin{equation}\label{eq:omega_int}
  \omega_{\mathrm{int}} = \sqrt{\max\!\bigl(|\theta|,\,10^{-7}\bigr)}.
\end{equation}
Because the square root uses $|\theta|$, the implementation always reports a real frequency magnitude even if $\theta$ changes sign during transients.

\section{Internal Time Scaling}

The implementation introduces a fixed internal time scale
\begin{equation}
  S = 20.
\end{equation}
This is an engineering modification intended to improve numerical conditioning for very low physical frequencies.

The parameter mapping used in the code is
\begin{align}
  a &= S\,\omega_{\mathrm{up}}, \\
  b &= S\,\omega_{\mathrm{up}}, \\
  k &= S\,k_{\mathrm{user}},
\end{align}
where $\omega_{\mathrm{up}}$ and $k_{\mathrm{user}}$ are user-facing constructor inputs.

If the physical sample period is $\Delta t$, the internal Euler step is
\begin{equation}
  \delta t = \frac{\Delta t}{S}.
\end{equation}

Likewise, the initializer rescales the user-supplied initial conditions by $S^2$:
\begin{equation}
  \theta_{\mathrm{stored}}(0) = S^2\,\theta_0,
  \qquad
  \sigma(0) = S^2\,\sigma_0.
\end{equation}

The physical output frequency in hertz is not $\omega_{\mathrm{int}}/(2\pi)$ directly; rather, it is
\begin{equation}\label{eq:physical_freq}
  f_{\mathrm{phys}} = \frac{\omega_{\mathrm{int}}}{2\pi S}.
\end{equation}
This matches the implementation of \texttt{getFrequencyHz()}.

\section{Exact Discrete-Time Update Sequence}

Let $y_n$ be the current sample, and let $x_{1,n}$, $\sigma_n$, and $\theta^{\mathrm{stored}}_n$ denote the values stored at the beginning of the sample. The implementation performs the following operations in this order.

\subsection{Filter Derivative}

First, the code computes the derivative of the low-pass state from the current stored state:
\begin{equation}\label{eq:x1dot_n}
  \dot{x}_{1,n} = -a\,x_{1,n} + b\,y_n.
\end{equation}

\subsection{Signal-Energy Weighting}

The implementation then forms an instantaneous signal-energy proxy
\begin{equation}\label{eq:signal_energy}
  E_n = x_{1,n}^2 + y_n^2 + 10^{-12},
\end{equation}
and a normalization factor
\begin{equation}\label{eq:gain_scaling}
  \rho_n = \frac{E_n}{E_n + 10^{-6}}.
\end{equation}
This factor suppresses aggressive adaptation when the signal is weak.

\subsection{Nonlinear Adaptation Term}

The auxiliary quantity
\begin{equation}
  \mathrm{tmp}_n = b\,y_n + a\,x_{1,n}
\end{equation}
is formed, followed by the nonlinear adaptation signal
\begin{equation}\label{eq:phi_n}
  \varphi_n
  = \dot{x}_{1,n}\,\mathrm{tmp}_n
    + x_{1,n}^2\,\theta^{\mathrm{stored}}_n.
\end{equation}

An important detail is that \eqref{eq:phi_n} uses the \emph{stored} value $\theta^{\mathrm{stored}}_n$ already present at the beginning of the sample. It does not use a freshly recomputed $\theta$ from the current sample before forming $\varphi_n$.

\subsection{Integrator Rate with Clipping}

The integrator rate is then computed as
\begin{equation}\label{eq:sigma_dot_n}
  \dot{\sigma}_n
  = \mathrm{clip}\!\bigl(-k\,\varphi_n\,\rho_n,\,-10^7,\,10^7\bigr),
\end{equation}
where
\begin{equation}
  \mathrm{clip}(u,\ell,h)=\max(\ell,\min(h,u)).
\end{equation}
This is a numerical safeguard added by the implementation.

\subsection{Fresh Algebraic Frequency Output}

Before integrating $x_1$ and $\sigma$, the code recomputes the algebraic estimate
\begin{equation}\label{eq:theta_out}
  \theta_n^{\mathrm{out}} = \sigma_n + k b x_{1,n} y_n.
\end{equation}

The internal angular-frequency magnitude and physical frequency outputs are then
\begin{align}
  \omega_{\mathrm{int},n}
  &= \sqrt{\max\!\bigl(|\theta_n^{\mathrm{out}}|,\,10^{-7}\bigr)}, \\
  f_{\mathrm{phys},n}
  &= \frac{\omega_{\mathrm{int},n}}{2\pi S}.
\end{align}

Thus, the reported frequency uses the freshly recomputed algebraic parameter~\eqref{eq:theta_out}, but the adaptation signal~\eqref{eq:phi_n} uses the previously stored $\theta^{\mathrm{stored}}_n$.

\subsection{Euler Integration of the Dynamic States}

The stored dynamic states are then advanced using explicit Euler with step $\delta t = \Delta t/S$:
\begin{align}
  x_{1,n+1} &= x_{1,n} + \dot{x}_{1,n}\,\delta t, \\
  \sigma_{n+1} &= \sigma_n + \dot{\sigma}_n\,\delta t.
\end{align}

After that, the implementation stores
\begin{equation}
  \theta^{\mathrm{stored}}_{n+1} = \theta_n^{\mathrm{out}}.
\end{equation}
Therefore, on the next sample, the adaptation law uses the previous sample's algebraic estimate.

\subsection{Phase Heuristic}

Finally, the code reports a heuristic phase estimate
\begin{equation}\label{eq:phase_est}
  \hat{\phi}_n = \operatorname{atan2}(x_{1,n+1},\,y_n).
\end{equation}
This phase output is not an independently validated analytic-signal phase; it is simply a practical proxy formed from the updated filter state and the current measurement.

\section{Interpretation of the Filtered State}

For a sinusoidal input
\begin{equation}
  y(t)=A\sin(\omega_0 t),
\end{equation}
the steady-state response of the first-order filter~\eqref{eq:x1_ode} is
\begin{equation}\label{eq:x1_sinusoid}
  x_1(t)
  =
  \frac{b}{\sqrt{a^2+\omega_0^2}}\,A\,
  \sin\!\bigl(\omega_0 t - \arctan(\omega_0/a)\bigr).
\end{equation}
Hence the phase lag of $x_1$ relative to $y$ is
\begin{equation}
  -\arctan(\omega_0/a).
\end{equation}
At the cutoff frequency $\omega_0=a$, this lag is exactly $-45^\circ$. It approaches $-90^\circ$ only in the high-frequency limit $\omega_0\gg a$.

Therefore, $x_1$ should not be described as an exact quadrature channel. It is more accurate to say that $x_1$ is a filtered, phase-lagged proxy for the input. The phase estimate~\eqref{eq:phase_est} becomes more interpretable only after the frequency estimate has settled and the operating point lies in a regime where the filter lag is appreciable.

\section{Practical Meaning of the Parameter $\theta$}

In continuous-time Aranovskiy-style observers, the adaptive parameter is commonly related to the squared sinusoidal frequency. The present implementation follows that spirit but uses a practical algebraic estimate with output law~\eqref{eq:omega_int}. For that reason, the code is best understood as tracking the \emph{magnitude} of the squared angular frequency through $|\theta|$, rather than as enforcing a strict sign convention at all times.

A negative initial value such as
\begin{equation}
  \theta_0 = \sigma_0 = -\frac{1}{4}\,\omega_{\max}^2
\end{equation}
is a sensible engineering initialization because it is consistent with the usual continuous-time target $\theta^\star \approx -\omega^2$. However, in the implementation the reported frequency remains real because of the absolute value and floor inside~\eqref{eq:omega_int}, not because $\theta_0$ is negative.

\section{Sampling and Nyquist Considerations}

The estimator runs on discrete samples, so the input must satisfy the usual Nyquist requirement. If the physical sampling interval is $\Delta t$, then the Nyquist frequency is
\begin{equation}
  f_N = \frac{1}{2\Delta t}.
\end{equation}
To avoid aliasing and false convergence, the signal of interest should lie comfortably below $f_N$, and anti-alias filtering should be applied before sampling if the source contains higher-frequency content.

Because the implementation is intended for low-frequency motion tracking, its internal time scaling improves conditioning, but it does not remove the need for adequate sampling.

\section{Sensitivity to DC Offsets and Slow Drift}

The implementation does not contain an explicit bias observer for the scalar input $y$ and does not perform high-pass filtering internally. As a result, DC offsets and very slow drift can bias both the filter state and the adaptation law.

This can be seen directly from the algebraic term $k b x_1 y$ in $\theta$ and from the dependence of the adaptation signal on both $x_1$ and $y$. If
\begin{equation}
  y(t) = y_0 + \tilde{y}(t)
\end{equation}
with nonzero constant component $y_0$, then products such as $x_1 y$ acquire bias terms that can distort the estimated frequency.

In practice, the estimator behaves best when the input is mean-removed or lightly high-pass filtered before frequency estimation.

\section{Numerical Safeguards in the Code}

The implementation includes several protective measures that are not part of a minimal textbook observer:
\begin{itemize}
  \item \textbf{Input validation:} non-finite samples are ignored, and updates with non-positive or non-finite $dt$ are skipped.
  \item \textbf{Energy weighting:} the factor $\rho_n$ reduces adaptation when the signal energy is very small.
  \item \textbf{Clipping:} $\dot{\sigma}$ is clipped to $[-10^7,10^7]$.
  \item \textbf{Square-root floor:} the frequency output uses $\max(|\theta|,10^{-7})$ to avoid singular behavior near zero.
  \item \textbf{Internal time scaling:} the fixed factor $S=20$ reduces stiffness for low-frequency operation.
\end{itemize}

These choices improve robustness in embedded and low-SNR settings, but they also mean the exact discrete-time implementation is not the same as an ideal continuous-time observer analyzed in the literature.

\section{Confidence and Lock Heuristics}

The implementation synthesizes a confidence score and a Boolean lock flag. These are \textbf{engineering heuristics} and are not part of the original Aranovskiy observer.

\subsection{Signal RMS Estimate}

A smoothed signal-power estimate is maintained from
\begin{equation}
  P_n = x_1^2 + y^2,
\end{equation}
using a one-pole smoother with time constant \texttt{rms\_tau\_s}. The reported RMS-like magnitude is then
\begin{equation}
  \mathrm{RMS}_n = \sqrt{\max(P_n^{\mathrm{smooth}},0)}.
\end{equation}

\subsection{Smoothed Frequency}

The physical frequency estimate is also smoothed with a one-pole filter using \texttt{freq\_smooth\_tau\_s}, producing a stability reference frequency.

\subsection{Confidence Target}

A target confidence is formed from two ingredients:
\begin{itemize}
  \item an amplitude term based on whether the smoothed RMS exceeds \texttt{lock\_rms\_min},
  \item a stability term based on how close the raw frequency is to the smoothed frequency.
\end{itemize}
The resulting confidence is then low-pass filtered with time constant \texttt{confidence\_tau\_s} and clipped to $[0,1]$.

\subsection{Lock Decision}

The lock flag is declared true only when all of the following hold:
\begin{itemize}
  \item the frequency lies within the configured physical range
        $[f_{\mathrm{lock,min}},f_{\mathrm{lock,max}}]$,
  \item the signal RMS exceeds \texttt{lock\_rms\_min},
  \item the filtered confidence exceeds \texttt{lock\_conf\_threshold}.
\end{itemize}

These quantities are useful diagnostics, but they should not be interpreted as formal convergence certificates.

\section{What This Implementation Does and Does Not Claim}

This implementation is best described as a lightweight nonlinear adaptive frequency tracker for low-frequency oscillatory signals. It is not an FFT estimator, and it is not a classical PLL. It uses a filtered state, a nonlinear adaptive integrator, and a practical algebraic mapping to frequency.

At the same time, this note does \emph{not} claim a formal Lyapunov theorem for the exact code as written, because the shipped implementation includes discretization, clipping, normalization, and heuristic outputs beyond the idealized continuous-time observer.

\section{Conclusion}

The provided C\texttt{++} implementation is a practical Aranovskiy-style frequency estimator specialized for robust low-frequency tracking. The exact algorithm is characterized by:
\begin{itemize}
  \item a first-order low-pass internal state,
  \item a nonlinear energy-weighted adaptation law,
  \item explicit Euler integration under internal time scaling,
  \item frequency output from the magnitude of an algebraic parameter,
  \item heuristic phase, confidence, and lock indicators.
\end{itemize}

This description matches the structure and update order of the code more faithfully than a generic continuous-time derivation.
